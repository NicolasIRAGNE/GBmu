◆ init_cpu()

int init_cpu ( struct gb_cpu_s *  gb,
struct rom_s *  rom,
enum gb_mode_e  mode 
)

Initializes the gb struct. This should be called after opening a rom but before any other function. This function will call the underlying functions appropriate to the Gameboy type.

Note
The struct gb_cpu_s* gb should be allocated by the caller.
The struct gb_cpu_s* gb is zeroed.
Parameters
gb
romThe rom to load. The rom has to be initialized before with open_rom().
modeThis parameter is used to specify the type of the Gameboy. It can be one of the following:
  • GB_MODE_DMG: Original Gameboy mode.
  • GB_MODE_CGB: Color Gameboy mode.
  • GB_MODE_GBA: Gameboy Advance mode. Not supported.
  • GB_MODE_AUTO: Auto-detect the gameboy mode. This is done by reading the ROM header.
  • GB_MODE_UNKNOWN: Unknown mode. This is not used but is here for completeness.
Returns
int 0 on success, non-zero on error.

Transfer data from memory to VRAM with an arbitrary length. This function is only available on CGB.

Parameters
gb
a8The lower 7 bits of a8 are the length of the transfer. In case of General transfer, the data is transferred immediately. In case of H-Blank transfer, 0x10 bytes are transferred each time the H-Blank interrupt is triggered.
mode0 for general transfer, 1 for H-Blank transfer.
Note
Starting a general transfer will interrupt any ongoing H-Blank transfer.

◆ request_interrupt()

void request_interrupt ( struct gb_cpu_s *  gb,
uint8_t  request 
)

Requests an interrupt by writing the corresponding value to the IF register.

Parameters
gb
requestThe interrupt request to set. Can be one of the following:
  • INT_VBLANK_REQUEST: Fired when the LCD screen finishes drawing the current frame.
  • INT_STAT_REQUEST: Generic interrupt fired when any of the events enabled by the stat register (STAT_OFFSET) occur.
  • INT_TIMER_REQUEST: Fired whenever the timer (TIMA_OFFSET) overflows.
  • INT_SERIAL_REQUEST: yeah i don't remember what this is for.
  • INT_JOYPAD_REQUEST: Fired whenever a button is pressed.
